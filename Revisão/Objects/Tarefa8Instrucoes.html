Fábrica de Refeições

Um restaurante contratou você para criar uma função para o site dele que permite definir uma refeição e um preço todas as manhãs para o Today's Special. Use seu conhecimento de getters e setters para garantir que qualquer pessoa que use a nova função possa gerar uma refeição e um preço para o Especial do Dia sem nenhum erro embaraçoso!

    Create the Menu Object

  1. Manteremos a refeição, o preço e seus respectivos getters e setters em um objeto chamado menu. Em app.js, crie um objeto de menu vazio.
  2. O objeto de menu manterá a refeição e o preço do Especial de Hoje como propriedades e eles não devem ser alterados diretamente. Dentro do objeto de menu, crie uma propriedade _meal com o valor de uma string vazia ('' ). Isso acabará por manter o nome da refeição.
  3. Em seguida, adicione uma propriedade _price com valor 0. Isso eventualmente manterá o preço da refeição e também não deve ser alterado diretamente.
  4. Sabemos que as propriedades que começam com _ não devem ser manipuladas diretamente. Mas apenas para validar esse conhecimento, vamos testá-lo! Abaixo do objeto de menu, manipule diretamente as duas propriedades atribuindo a _meal um valor numérico e _price um valor de string. Em seguida, abaixo das novas atribuições, console.log() o objeto de menu para ver como não verificar esses valores pode causar confusão para um visitante do site!

    5. Adicionando Métodos Setter

  5. Para reatribuir com segurança as duas propriedades do menu, podemos adicionar setters que verificam os valores que estão sendo atribuídos. Abaixo das propriedades, use a palavra-chave set para criar um método setter de refeição com mealToCheck como parâmetro. Deixe o corpo da função vazio por enquanto.
  6. No corpo do método setter, crie uma instrução if que verifique se mealToCheck é uma string. Se for, retorne a propriedade _meal do objeto com mealToCheck atribuído como o valor.
  7. Usando os mesmos procedimentos acima, use uma palavra-chave definida para criar um setter de preço com priceToCheck como parâmetro. Esse método deve garantir que o valor associado a _price seja sempre um número.
  8. Abaixo do objeto de menu, defina os valores de _meal e _price usando os métodos setter recém-criados. Em seguida, console.log() no objeto de menu para verificar sua funcionalidade.

    9. Adicionando Método Getter

  9. Agora é hora de retornar com segurança os valores das propriedades _meal e _price de forma legível. Em vez de acessar diretamente as propriedades, podemos usar um método getter que verifica proativamente se uma refeição e o preço foram definidos corretamente, antes de retornar os valores. Abaixo dos setters, use a palavra-chave get para criar um método todaysSpecial. Deixe o corpo da função vazio por enquanto.
  10. No corpo do getter, crie uma instrução if…else para verificar se os valores _meal e _price existem (ou são valores verdadeiros). Em caso afirmativo, retorne uma string informando aos possíveis visitantes do site qual é o Especial de hoje. Por exemplo: “O especial de hoje é espaguete por R$ 5!” Se os valores _meal e _price não existirem (ou forem falsos), retorne a string 'Refeição ou preço não foi definido corretamente!'.

    11. Get Especial do Dia

  11. Por fim, use o método getter para console.log() Especial do dia. Supondo que você tenha usado o setter de refeição para atribuir uma string e o setter de preço para atribuir um número na tarefa 8, você deverá ver o Especial de hoje registrado no console. Se você deseja estender seu aprendizado neste projeto, tente adicionar uma variedade de refeições e preços para definir aleatoriamente e obter o Especial de hoje!
